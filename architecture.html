GPU Architecture

Core components that help GPUs execute parallel tasks efficiently.

⚙️ Streaming Multiprocessors

Clusters of cores execute many threads in parallel and schedule work dynamically.

🧩 Memory Hierarchy

Registers, shared memory, cache, and VRAM balance speed and capacity for performance.

🔆 Compute Pipelines

Dedicated units for geometry, rasterization, shading, and modern ray tracing workloads.

Execution Flow (Simplified)

  1. Task enters command queue.
  2. Scheduler assigns work to multiprocessors.
  3. Thousands of cores process data in parallel.
  4. Results are written to memory or display output.

CPU vs. GPU Architecture

Feature Central Processing Unit (CPU) Graphics Processing Unit (GPU)
Core Count Few (8–24 powerful cores) Thousands (10,000+ small cores)
Optimization Serial Processing (one by one) Parallel Processing (all at once)
Latency Low (fast individual tasks) High (high throughput for big data)

Data Bottlenecks

Memory Hierarchy: Speed vs. Size

Performance relies on moving data quickly between these layers.

Registers (Internal Core Memory) Fastest / Smallest
L2 Cache / Shared Memory High Speed / Medium Size
VRAM (GDDR6X) Standard Speed / Massive Size
